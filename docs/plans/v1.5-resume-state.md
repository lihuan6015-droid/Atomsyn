## 🐛 收尾密集 bug 修复 (Session 4 集中)

这些都是在 V1.5 基本功能 ship 之后,实测时暴露的"看起来对但运行不 work"类问题。已全部沉淀进 `atom_exp_macos-tauri-hit-test-vs-drag-region_1775656757`:

1. **Settings 白屏 v1**: `config.activeProvider` 缺失 → 防御式三级降级
2. **Settings 白屏 v2**: `config.copilot` 缺失 → useEffect 加载阶段整体 normalize 所有字段
3. **Tauri 拖动只能拖一次**: macOS Overlay 模式的红绿灯 hit-test 区接管 → 放弃 Overlay,改用 native title bar
4. **Native title bar 跟系统外观不跟 app**: tauri.conf 加 `theme: "Light"` + 运行时 `setTheme()` + onRehydrateStorage sync 三件套
5. **`setTheme()` 调用静默失败**: Tauri v2 deny-by-default,缺 `core:window:allow-set-theme` 权限 → 补 capabilities
6. **Splash 在浅色主题下文字看不见**: 改 `bg-[#0a0a0b]` 为 outer wrapper + 内层 `hero-gradient`,所有色值加 `isDark ? ... : ...` 双分支
7. **Splash 自动进入太快**: 改 `autoAdvance` 默认 false + 加 min-hold 450ms/步
8. **scan-skills 漏掉符号链接**: walkSkillFiles 跟随 `isSymbolicLink()` + cycle guard,从 64 → 98 skills
9. **打开文件夹"打开失败"**: 不走 shell plugin scope,新 Rust `open_path` 命令直接 shell 出去
10. **Vite + Rust 数据目录分叉**: Tauri `app_data_dir()` 自动加 bundle id → Rust 写到 `com.circlelee.ccl-atlas/ccl-atlas` 而 vite 写到 `ccl-atlas` → 两边各说各话。手工构造平台路径完全对齐
11. **首次启动 seed-check 永远不弹**: vite seedDataDir 没写 `.seed-state.json` → 改成 seed 完毕后自动写一份初始 manifest
12. **GUI 看不到 experience atoms**: vite 数据目录硬编码到项目 /data,而 atlas-cli 写到 `~/Library/...`。修复在 (10),GUI 现在和 CLI 读同一处
13. **`tsc -b` build 失败**: `tsconfig.node.json` 缺 `target: ES2022` → Set 迭代报错 → 阻塞 tauri build
14. **更新经验只能 write+rm**: 新增 `atlas-cli update --id` 命令,原子按 id 定位 + merge + slug 迁移 + rmdir 旧目录,冷启动新会话也能用

你正在为一个叫 **CCL Atlas** 的本地优先桌面应用完成 V1.5 版本。核心使命:

## 💡 Session 1 发现的 Tribal Knowledge (非显而易见)

1. **Claude Code 热加载 skills**: 写到 `~/.claude/skills/` 的新 SKILL.md 不需要重启 session 就立即被 Skill tool 发现. T-2.3 的 atlas-read SKILL.md 写完后, 理论上当前 session 就能调用. (从 Session 1 的 atlas-write-spike 首次出现的时机实测验证.)

2. **Tauri 第一次编译已缓存**: `cargo check` 已跑过一次, Tauri v2 的 ~500MB 依赖都下载完成. 后续 `tauri:dev` / `tauri:build` 会非常快 (估计 1-3 分钟, 不是首次的 10+ 分钟).

3. **`ajv` 已装但未使用**: `package.json` 有 `ajv` + `ajv-formats` 但 src/ 里没有任何代码 `import` 它. 所以 `skills/schemas/*.json` 在运行时不是被 validator 消费的, 它们是文档. 类型真相源是 `src/types/index.ts`. 这让 schema 改动不需要担心 runtime 炸.

4. **V1 用 Vite dev plugin 做 API, 不是 Express**: `vite-plugin-data-api.ts` 注入 `/api/*` 端点. Tauri 模式下需要 future refactor 让 `src/lib/dataApi.ts` 根据 `isTauri()` 切换到 `@tauri-apps/api/fs` 直接读磁盘 (不是 HTTP). **这不是 V1.5 必须, V1.5 可以让 Tauri 窗口内也跑 Vite dev server, 暂时保持 /api 架构, 后续重构**.

5. **现有 `scripts/rebuild-index.mjs` 和 `vite-plugin-data-api.ts` 的 `loadAtoms` 会 walk 整个 `data/atoms/` 并 map 假设 methodology shape**. Session 1 已加 `kind === 'methodology'` 过滤防止崩溃. 但若 Sprint 3 要给 experience/skill-inventory 也建索引 (用于 Skill 地图搜索 or Agent Feed 快速查询), 需要扩展 KnowledgeIndex 类型 + 添加 `experiences` / `skillInventory` 新字段 + 更新两处 loader.

6. **数据目录双模式并存**:
   - Dev 模式 (CCL_DEV_DATA_DIR 或 npm run dev) → 项目 `/data/` (git-tracked, 用于 dogfooding)
   - Tauri 模式 → `~/Library/Application Support/ccl-atlas/` (首次启动创建, 分发用)
   - 用户在同一台 Mac 可能同时有两套数据. Settings UI 的"当前数据目录"显示必须清楚标注是哪一套.

7. **Experience atom 的 slug 从 name 派生**: atlas-cli 的 `deriveSlug()` 把 name 转 kebab-case, strip accents, 截取 40 字符. **中文 name 会变成空字符串 → fallback 到 "atom"**. 这是一个 V1.5 bug/limitation. 如果 Sprint 3/4 时发现中文 experience atoms 都堆在同一 slug 文件夹, 需要改 derive 逻辑 (比如从 id 派生而非 name).

8. **atlas-cli 的 read 命令只扫 experience/**, 不扫 methodology 或 skill-inventory. 这是设计决策: methodology 由 GUI 主导 (L1), experience 由 agent 主导 (L2), skill-inventory 是 L1 只读 (不进 agent 上下文). Sprint 3 的 read 升级可考虑加 `--include-methodology` flag 让 agent 在适当场合也能 surface methodology atoms.

9. **第一张 experience atom 的特殊性**: `data/atoms/experience/claude-skill-mcp-server/atom_exp_claude-skill-over-mcp-for-local-sovereignty_1728443200.json` 是 Session 1 自己产出的, 记录的是"为什么选 Claude Skill 而不是 MCP". 这是一个递归闭环——CCL Atlas 用自己记录自己的架构决策. **不要删掉, 它是 V1.5 的第一个 dogfooding 证据**.

10. **主 agent 上下文策略**: Session 1 结束时主 agent 上下文用到 34%. 剩余 V1.5 工作如果集中做, 预计还要 30-40% (UI 任务派 subagent 承担大头). 如果中途触发 autocompact, 新 "thread" 可以从读本文档开始.
