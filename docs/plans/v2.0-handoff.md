## 🎯 V1.5 的核心产品哲学 (V2.0 必须继承)

这些是 V1.5 framing session 沉淀的不可动摇的设计 DNA。V2.0 的任何新功能都必须**先回到这里检查不冲突**:

1. **本地 100%, 主权第一**: 用户数据从不离开本地。无云同步 (V1.x) / 无遥测 / 无账号。LLM key 存 localStorage 永不进 git。
2. **L1 + L2 双层缺一不可**: 只做 L1 = 又一个 Notion (资产是死的)。只做 L2 = 又一个 RAG/MCP 工具 (用户感受不到主权)。两层互锁才能形成"使用即维护"的飞轮。
3. **大厂结构性不会做**: 这不是窗口期,是**结构性安全的小生境** (反规模化 + 反数据飞轮 + 哲学方向相反)。
4. **使用即维护**: 资产在被 agent 调用的同时被自动更新和成长。用户的 FOMO 囤积循环必须被这个机制打破。
5. **CLI-first**: agent 永远不直接写磁盘。所有 schema 演进、路径解析、锁冲突、索引重建都由 atlas-cli 集中承担。
6. **设计语言硬约束**: Linear/Raycast 变体 + Inter + 玻璃态 + spring + 4 层渐进披露 + 浅色优先 + 全暗色对照。任何 UI 任务必须先读 mockups + 派 code-reviewer 审查。

## 🐛 V1.5 最重要的两条元教训 (V2.0 不要重复踩)

### 元教训 1 · Tauri v2 deny-by-default 而且静默失败

凡是前端调 Tauri window/fs/shell API 没反应:
- ❌ 第一反应不要去查 React 代码或 TypeScript
- ✅ 第一步打开 `src-tauri/capabilities/default.json`,确认对应 `core:xxx:allow-yyy` 权限是否已 grant
- ✅ 第二步看 `src-tauri/tauri.conf.json` windows 配置是不是组合错了 (比如 Overlay 模式的 hit-test 副作用)
- ✅ **永远不要 `catch(() => undefined)` Tauri 的 promise 错误**,deny-by-default 拒绝本来就隐身,你再 swallow 一层就完全无法 debug

### 元教训 2 · 三端数据路径必须字符级别一致

vite 插件 / Rust 命令 / atlas-cli 三个端写同样数据目录的字符串构造逻辑必须**完全镜像**。Tauri 的 `app.path().app_data_dir()` 会自动加 bundle id 前缀,**不要用它**,改用 `home_dir() + cfg!(target_os)` 手工构造。**任何路径分叉都会让 GUI 和 CLI 各说各话**,debug 极度困难。
