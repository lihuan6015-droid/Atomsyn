## ⚠️ V2.0 新增的 2 条元教训(继承 V1.5 的两条之上)

### 元教训 3 · 大型重命名的 grep 模式必须覆盖所有形态
V1.5 的 2 条元教训(Tauri 权限静默失败 + 三端路径字符级镜像)在 M0 第一次覆盖时**漏了** `CCL Atlas`(带空格的字型)这一类,因为我的初始 grep 模式只覆盖 `ccl-atlas` / `ccl_atlas` / `atlas-cli` 等 kebab/snake/dash 形式。后来在 src/components/splash, TopNav, index.html 等 11 个文件发现遗留。

**教训**: 任何跨大型代码库的 rename,grep 模式必须显式包含:
- `kebab-case` (ccl-atlas)
- `snake_case` (ccl_atlas)
- `PascalCase` (CclAtlas)
- `Title Case with space` (CCL Atlas) ← **最容易漏**
- `UPPER_CASE` (CCL_DEV_DATA_DIR)
- 各种带点前缀 (.ccl-atlas, .ccl-atlas-config.json)

### 元教训 4 · scan-only 工具必须有 "sync" 模式
`scan-skills.mjs` 原设计只做"create/update",从不删除。长期运行后用户的 inventory 和磁盘实际 skill 会慢慢偏离。**凡是"索引"类脚本都应该同时具备 seen-set 跟踪 + 孤儿 prune 模式**,否则迟早产生"幽灵条目"污染 GUI。
