# V1.5 Sprint 3 Design Language Review
**Date**: 2026-04-08
**Reviewer**: code-reviewer agent

## Summary

Sprint 3 + splash deliverables hit the design contract on the major beats: glass surfaces, brand gradient (violet/sky/emerald), spring easing, hero-gradient hero blocks, progressive disclosure on the skill cards, and Inter/JetBrains-Mono usage are all consistent with `docs/mockups/atlas.html`. Dark-first parity is broadly correct in the in-app pages. The biggest risks are (1) **the splash screen is hard-coded to dark** with no light-mode parity, (2) **three independent `hashColor` agent-chip palettes** (AgentActivityFeed, AtomDetailPage, SettingsPage) drift in colors and ring/border style — the same agent will look different in three places, and (3) **the AgentActivityFeed drawer hard-codes a white background** that breaks dark mode. The Skill 地图 page uses raw emoji glyphs in filter/action buttons rather than `lucide-react` icons, which is the only contract-level deviation in that file. Nothing in the new TopNav tab, AtomDetailPage lock badge, or SettingsPage Agent Permissions section is broken — they're polishable but ship-quality.

- (lines 295–297) The frontmatter `<pre>` block is good (mono font, neutral-50 surface), but the JSON literal is interpolated with leading whitespace from JSX indentation, so the rendered output has a stray indent on line 1. Cosmetic.
- (line 231) `style={{ ['--glow-color' as any]: meta.colorVar }}` — the `as any` cast is harmless but `as React.CSSProperties` or a typed helper would be cleaner.
- The L1 hover-reveal meta row (lines 267–283) opacity-transitions, but on touch / keyboard-focus there's no equivalent reveal. Minor a11y nit.

Must-fix: none for ship, but the emoji-vs-lucide inconsistency is the closest thing to a contract violation in this file.

## Global observations

1. **Three independent agent-chip color palettes**. This is the only real cross-file inconsistency in the sprint:
   - `AgentActivityFeed.tsx` lines 24–34 — 4 colors, `bg-*-500/10 text-*-600 dark:text-*-400 border-*-500/20`
   - `AtomDetailPage.tsx` lines 776–786 — 4 colors, identical to above (this one is OK)
   - `SettingsPage.tsx` lines 540–551 — 5 colors (adds pink), `bg-*-500/10 text-*-700 dark:text-*-300 ring-*-500/30`
   The first two agree; SettingsPage drifts. The same agent name (e.g. `claude-code-cli`) will render with a different hue index AND a different text shade AND a ring instead of a border depending on which page you're on. **Recommendation**: extract a single `agentChipClass(name)` utility into `src/lib/agentColors.ts`, import it in all three places. This is the highest-leverage fix in the whole review.

2. **Splash screen is dark-only**. The dark-first contract is "dark-first, **full light parity**." The splash hard-codes dark text and white-alpha borders, with no `dark:` light counterparts. If the user's persisted theme is light, the splash → app handoff is jarring. Either commit to "splash is always a dark loading veil" (and remove the false dark-mode-only comment in the code) or wire it to the theme.

3. **Emoji vs lucide icons**. The codebase mostly uses lucide. SkillMapPage is the outlier — it uses lucide in some places (card header glyphs, line 250) and raw emoji in others (filter buttons line 192, action buttons lines 364/378/392, empty state line 214). Pick one.

4. **Animation library mixing**. AgentActivityFeed and SplashScreen use CSS keyframes with explicit `cubic-bezier(0.16, 1, 0.3, 1)`. SkillMapPage Toast uses `tailwindcss-animate` utilities (`animate-in fade-in slide-in-from-bottom-4`). Atom card collapse uses the `.collapse-content` grid-rows trick. Three different animation idioms in one sprint. Not a contract violation per se, but worth standardizing in V1.6.

5. **Border opacity drift**. Most files use `border-neutral-200/60 dark:border-neutral-800/60`. AgentActivityFeed and parts of AtomDetailPage use `/70`. SettingsPage uses `/70`. Cosmetic, but a single token (e.g. `border-app-divider`) would prevent further drift.

6. **Scrollbar consistency**. AgentActivityFeed feed list (line 217) is missing `scrollbar-subtle` while the sister scroll surfaces in AtomDetailPage and SettingsPage have it.

If staying dark-only, remove the false claim of light parity from CLAUDE.md OR add an `aria-hidden` comment block documenting the intentional dark veil. If supporting light, add `dark:` counterparts to every hard-coded `text-neutral-100`, `border-white/5`, and `text-amber-300` style. (`src/components/splash/SplashScreen.tsx` lines 65, 105, 140–149; `src/components/splash/InitStepRow.tsx` lines 47, 65, 82.)

It overrides the dark glass surface and produces a white drawer over the dark app. Replace with the standard `bg-white/70 dark:bg-[#0a0a0b]/70` class pair.

3. **(should-fix)** Extract a single agent-chip color helper. Create `src/lib/agentColors.ts` exporting `agentChipClass(name: string): string` and import it in:
