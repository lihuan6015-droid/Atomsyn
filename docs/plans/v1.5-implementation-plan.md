### 1.2 北极星 Demo 四幕（唯一验收标准）

| 幕 | 场景 | 主语 | 通过条件 |
|---|---|---|---|
| 一 | 在 Claude Code 里说"沉进去" → atom 自动入库 | AI | experience-atom 文件真实落盘，schema 正确 |
| 二 | 换上下文打开 Claude Code → read skill 自动注入 | AI | 上下文注入后 Claude 表现"懂我"（盲测 7/10）|
| 三 | 周日打开 Atlas 桌面 app → 看见过去一周活动 → 校准 1-2 张 | 人 | Agent Feed 有真实数据；校准操作可写回 |
| 四 | 打开"Skill 地图"Tab → 卡片化看见本地 40+ skill | 人 | 扫描结果真实；agent 辅助摘要能生成 |

**任何一幕跑不通 = V1.5 未完成。**

### 1.3 设计语言硬约束 ⭐（用户明确强调）

**每一个 UI 相关任务都必须：**
1. **先读** `docs/mockups/atlas.html` 和 `docs/mockups/atom-card.html` 作为视觉 source of truth
2. **继承** `src/index.css` 中的 CSS 变量（阶段色、圆角、spacing scale、spring 缓动）
3. **复用** `src/components/` 里已有的组件 pattern（不自己造轮子）
4. **遵循** CLAUDE.md 里的视觉契约：
   - 风格：**Linear / Raycast variant of Apple/Google**（现代、玻璃态、spring 动画）
   - 字体：**Inter + JetBrains Mono**
   - 动画：Framer Motion 或 CSS spring `cubic-bezier(0.16, 1, 0.3, 1)`
   - 渐进披露 4 层模型
   - 暗色优先 + 亮色全奇偶
5. **每个新 UI 组件的 PR 必须附**：暗色截图 + 亮色截图 + 与 mockup 的 diff 说明

**违反设计语言 = 任务不通过。** 任何 subagent 如果没法访问 mockup 文件，主 agent 必须把 mockup 相关片段贴进 task packet。

### 1.4 数据目录策略（B3）

```
默认路径（Tauri 运行时）:
  macOS:   ~/Library/Application Support/ccl-atlas/
  Windows: %APPDATA%/ccl-atlas/

开发模式路径（npm run dev）:
  项目根 /data/  (V1 现状，保持不变)

用户自定义:
  Settings → 数据目录 → 支持手动切换 + 迁移向导
  存储在 ~/.ccl-atlas-config.json（主配置，和数据分离）
```

**切换机制**：启动时读 `~/.ccl-atlas-config.json` → 如果没有 → 默认路径 → 如果指定了 → 用指定路径。Dev 模式下优先使用项目内 `/data/`（env var 控制）。

### 1.5 Context / Subagent 策略

| 任务性质 | 执行者 |
|---|---|
| 战略决策 / Schema 设计 / 架构调整 / scope 变更 | **主 Agent**（需要全局上下文）|
| 单文件组件实现 / 工具脚本 / Tauri 配置 / 测试 | **general-purpose subagent**（原子任务）|
| 设计 diff 审查 / 代码质量审查 | **code-reviewer subagent** |
| API 文档查询 / 库选型研究 | **Explore subagent** 或 **context7 MCP** |

**派发原则**：
1. 每个 subagent 任务包必须自包含：引用本文档的 task ID + 贴关键 schema + 指定输出路径
2. Subagent 返回后主 agent 必须验收（读文件 + 跑 lint + 核对验收标准）才能标记 done
3. 任何 subagent 提出的 scope 变更都必须回主 agent 裁决，不能自作主张

### 1.6 范围蠕动防火墙（铁规）

### T-1 · Gap-1 · Schema 扩展

#### T-1.1 · atom schema 加 `kind` 字段 + 新增两类 schema
- **依赖**: T-0.1 通过
- **执行**: 主 Agent（涉及产品决策）
- **输入**:
  - 现有 `skills/schemas/atom.schema.json`
  - 现有 `src/types/index.ts`
- **工作内容**:
  1. 给 `atom.schema.json` 加顶层字段 `kind: "methodology" | "experience" | "skill-inventory"`，默认 `methodology`
  2. 为 `experience` 和 `skill-inventory` 分别新建 schema（可以复用 base + 差异字段）
     - `experience-atom`：轻量，字段包括 `sourceAgent`、`sourceContext`、`insight`、`codeArtifacts`、`screenshots`
     - `skill-inventory-item`：字段包括 `localPath`、`toolName`（claude/cursor/codex）、`frontmatter`、`aiGeneratedSummary`、`aiGeneratedTags`、`userMarked`（常用/已遗忘/未分类）
  3. 更新 `src/types/index.ts` 对应 TypeScript 类型
  4. 所有现有 V1 atom 文件的 `kind` 字段补默认值（运行一次 migration 脚本 `scripts/migrate-v1-to-v1.5.ts`）
- **交付物**:
  - `skills/schemas/atom.schema.json`（更新）
  - `skills/schemas/experience-atom.schema.json`（新）
  - `skills/schemas/skill-inventory-item.schema.json`（新）
  - `src/types/index.ts`（更新）
  - `scripts/migrate-v1-to-v1.5.ts`（一次性脚本）
- **验收标准**:
  - 所有现有 V1 atom 通过新 schema 验证
  - `npm run lint` 无错
  - 运行 migration 后，现有 atom 全部有 `kind: "methodology"`
- **预计工时**: 30 分钟
- **context 开销**: 低

### T-5 · Gap-5 · stats 自动更新

#### T-5.1 · usage-log.jsonl 写入 + atom stats 同步
- **依赖**: T-2.2 + T-2.4
- **执行**: **general-purpose subagent**
- **工作内容**:
  1. atlas-cli 的 write/read 操作都追加一条 JSONL 到 `data/growth/usage-log.jsonl`
  2. 日志 schema：`{ts, action: 'write'|'read', atomId, agentName, sessionId?, queryContext?}`
