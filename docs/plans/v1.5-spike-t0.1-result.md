## 🎉 意外收获（Bonus Findings）

### Finding A · Claude Code 热加载 skills
**发现**：写完 `~/.claude/skills/atlas-write-spike/SKILL.md` 后的**第一次**后续交互，系统 reminder 里立即列出了 `atlas-write-spike` 这个 skill——**不需要重启 session**。

**影响**：这比我们预期的好。意味着 V1.5 的 Gap-2 实现可以支持"**热插拔**"式的 skill 更新体验：
- 用户修改 `atlas-write/SKILL.md` 后不需要重启 Claude Code
- 我们可以在 GUI 的 Settings 里提供"重新生成 skill 文件"按钮，即改即生效
- Dogfooding 迭代速度会显著加快

### Finding B · Skill 形态与"资产=文件"哲学完全契合
**发现**：整个 spike 过程中，skill 就是一份 markdown 文件，skill 的"能力"就是 Claude 按照 markdown 指令用 allowed-tools 干活。整个链路**没有服务、没有进程、没有状态**。

**影响**：进一步确认了 S2.2 决策的正确性。V1.5 的 `atlas-write` 和 `atlas-read` 只需要是两份 markdown + 一个可选的 helper CLI（用于 schema 严格校验和数据目录路由），总实现量可能比估算的还要少 20-30%。

### Finding C · spike 产物本身就是 V1.5 的第一张 experience atom
**发现**：spike 的"沉淀素材"是 S2.2 架构决策——这是一个真实的、高价值的知识资产。意味着 V1.5 T-1.1（schema 扩展）完成后，我们可以**把这张 spike atom 直接作为 V1.5 的第一张 experience atom 导入**，省去造 fixture 的时间。

**动作**：在 T-1.1 完成后，把 `/tmp/atlas-spike-output.json` 迁移到 `data/atoms/experience/v1.5-scoping/claude-skill-over-mcp.json`（路径按最终的 experience 目录结构调整）。

## ⚠️ 已识别的 Caveats（需要在 T-2.1 处理）

### Caveat 1 · 触发精度
Spike 里我是手动按指令执行的，没有测试"**用户说'沉进去'时 Claude 会不会自动选中 atlas-write 这个 skill**"。因为 description 写得够具体（列出了多种中英文触发词），理论上 Claude 的 skill dispatcher 应该能命中，但**需要在 T-2.5 的 e2e 测试里用真实对话验证一次**。

**缓解**：T-2.5 测试时用一个干净的 Claude Code 会话，说"把我们刚聊的这个 Tauri 坑沉成一条经验"，观察 dispatcher 是否主动调 atlas-write。如果不命中，需要优化 description。

### Caveat 2 · Insight 幻觉风险
Spike 里的 insight 是我（主 agent）亲手写的，所以信息保真。但**当真正在 Claude Code 会话里被触发时**，Claude 需要"回忆"对话上下文——这里有幻觉风险，可能会补造实际没讨论过的步骤。

**缓解**（T-2.1 要做）：
- 在 SKILL.md 指令里加一条强约束："只沉淀本 session 实际出现过的内容，未出现的细节标记为 `<待补充>`"
- helper CLI 加一个 `--strict` 模式，要求 insight 必须引用 session 中的原句（至少 1 处）
- V1.5 T-6.2（atom 校准）提供"编辑"入口，用户周日巡检时可以手动修正幻觉

### Caveat 3 · 并发写入未测试
如果用户在两个 Claude Code 会话里同时触发 atlas-write，文件系统可能产生竞态。

**缓解**：T-2.2 实现 helper CLI 时用文件锁（fcntl 或 atomic rename），V1.5 先做简单的最后写入获胜 + 写入前检查 mtime。

### Caveat 4 · Schema 演化
Spike 的 schema 是为 V1.5 简化过的（没有 bookmarks / stats / example 等 V1 methodology-atom 的字段）。T-1.1 时需要确认 experience-atom 的完整 schema。

**缓解**：T-1.1 就是专门做这件事，没有阻塞。
