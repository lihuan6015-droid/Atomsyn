## 场景: 方法论与经验碎片在 AI 调用中相遇

### Step 1 · 沉淀 (atomsyn-cli ingest)

```bash
echo '{"title":"用户访谈发现产品创新24步法中趋势扫描的盲区",
  "summary":"在进行客户访谈时发现,大多数PM在做趋势扫描时只关注技术趋势,忽略了文化和社会层面的变化...",
  "role":"产品","situation":"访谈","activity":"分析",
  "insight_type":"反直觉",
  "tags":["趋势扫描","客户访谈","文化盲区","产品路线图"],
  "confidence":0.85}' | atomsyn-cli ingest --stdin
```

**结果**: 
- Fragment `atom_frag_24_1775800461069` 写入成功
- 自动关联 5 个方法论: `atom_jtbd`, `atom_macro_scanning`, `atom_day_in_the_life`, `atom_vpc_canvas`, `atom_demand_supply_canvas`
- **无需 LLM 调用**, 纯关键词匹配完成关联

### Step 2a · 读取认知地图 (atomsyn-cli read)

```bash
atomsyn-cli read --query "趋势扫描 文化" --top 3
```

**结果**:
- 方法论命中 3/125: PEST-C 分析, 宏观趋势扫描, 数据假设校验法
- **宏观趋势扫描的 📎 列显示 1** — 标记有用户碎片
- 经验碎片直接命中 1 条 (刚才写入的)
- 认知全貌: 125 方法论 + 11 碎片 (产品×4)

### Step 2b · 钻取详情 (atomsyn-cli get)

```bash
atomsyn-cli get --id atom_macro_scanning
```

**结果**:
- 返回完整方法论 markdown (核心理念 + 步骤 + 案例)
- 末尾自动附加 📎 碎片: "用户访谈发现...趋势扫描的盲区"
- 输出 ≤ 8000 字符

### Step 3 · Agent 自然融入

Agent 在回答用户关于趋势扫描的问题时,可以自然说出:
> "按你知识库里的'宏观趋势扫描'方法论,这里应该先圈定扫描边界...
> 你之前在访谈中也发现过类似的问题 —— 大多数 PM 在做趋势扫描时 90% 精力在技术趋势上,完全忽略了文化维度(来自 `atom_frag_24_1775800461069`)。
> 建议至少用 20% 的精力在社会/文化维度。"

## M4 架构总结

```
写入端 (atomsyn-write skill)
  ↓ atomsyn-cli ingest --stdin
  ↓ 自动关键词匹配 → linked_methodologies 填充
  ↓ 写入 atoms/experience/fragment/<slug>/
  
读取端 (atomsyn-read skill)
  ↓ atomsyn-cli read → 认知地图 (紧凑表格 + 📊 全貌)
  ↓ atomsyn-cli get --id → 方法论详情 + 📎 关联碎片
  ↓ Agent 自然融入回答
```
