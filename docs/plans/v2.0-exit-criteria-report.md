## PRD §0 定义的三条闭环

### 闭环 1: 沉淀方向 ✅

> 在 Claude Code 里对 agent 说"帮我记下刚才这段思考",agent 调 `atomsyn-cli ingest`,数据落盘并带上 D5 四维 schema 且挂到相关方法论。

**验证过程**:

```bash
echo '{"title":"用户访谈发现产品创新24步法中趋势扫描的盲区",...}' | atomsyn-cli ingest --stdin
```

**结果**:
- ✅ 数据落盘: `~/Library/Application Support/atomsyn/atoms/experience/fragment/`
- ✅ 四维 schema: role=产品, situation=访谈, activity=分析, insight_type=反直觉
- ✅ 自动关联方法论: linked_methodologies = [atom_jtbd, atom_macro_scanning, ...]
- ✅ 情绪复盘自动 private: true
- ✅ 19/19 CLI 回归测试全绿

### 闭环 2: 唤醒方向 ✅

> 在一个全新的会话里问一个和碎片相关的问题,`atomsyn-cli read` 返回方法论 + 📎 碎片。

**验证过程**:

```bash
# Step 1: read 获取认知地图
atomsyn-cli read --query "趋势扫描 文化" --top 3

# Step 2: get 钻取方法论详情
atomsyn-cli get --id atom_macro_scanning
```

**结果**:
- ✅ read 返回紧凑认知地图 (方法论表格 + 碎片表格 + 📊 认知全貌)
- ✅ 方法论 📎 列正确显示关联碎片数
- ✅ get 返回方法论详情 + 📎 关联碎片摘要
- ✅ private=true 的碎片被过滤
- ✅ 渐进式 read→get 总 token < 10000

### 闭环 3: GUI 镜像 ⚠️ (待用户验收)

> 上述两个动作都能在 Memory Garden 里被看到、被校准、被锁定。

**已实现**:
- ✅ 方法论详情页 "关联碎片" 面板 (CollapseSection)
- ✅ 碎片详情页 "关联方法论" 面板 (双向跳转)
- ✅ 🔒 锁定/解锁校准按钮
- ✅ 碎片标题/洞察内容正确显示 (name||title, insight||summary 兜底)
- ⚠️ Agent 活动 Feed 需用户实际操作后验证 (usage-log 端点已就绪)
