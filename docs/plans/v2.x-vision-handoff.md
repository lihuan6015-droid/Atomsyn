### 价值主张 (V2.1 确认)

> **让你积累的认知，在需要时醒来。**
>
> 方法论、踩坑、顿悟、阅读——它们没有消失，只是沉睡在你的记忆角落。Atomsyn 把它们变成有结构、会生长的认知地图。
>
> 它会告诉你哪些领域已经扎实、哪些还是空白，在你懒得回头看的时候轻轻推你一把。
>
> 而当你下一次和 AI 对话时，你曾经的某个顿悟会安静地出现在回答里——不是被搜索出来的，是它自己找到了你。
>
> *Atomsyn — it remembers, so you can grow.*

### 产品定位

**认知双向操作系统** — 底层仓库 + 中间结构层 + 上层教练层，三层互相喂养。

```
┌─────────────────────────────────┐
│  上层 · 教练层 (Coach)           │  ← 主动引导：看见盲区、推动复盘、建议行动
│  导师模式 · 认知雷达 · AI 复盘    │     ✅ P1 已交付
├─────────────────────────────────┤
│  中间 · 结构层 (Structure)       │  ← 让认知"长出骨架"：方法论 × 碎片相遇
│  多骨架 · 书架提炼 · 统计可视化   │     ✅ P0 已交付
├─────────────────────────────────┤

│  Agent CLI · 笔记模块 · 记忆花园  │     ✅ V2.0 已交付
└─────────────────────────────────┘
```

## 3 · ✅ 认知洞察 — AI 复盘分析 (P1 已交付)

### 已完成内容

**后端分析 API**（`vite-plugin-data-api.ts` + `scripts/lib/analysis.mjs` 共享引擎）：
- `GET /api/analysis/dimensions` — 四维度聚类（role/situation/activity/insight_type）+ role×situation 交叉矩阵 + 时间衰减
- `GET /api/analysis/timeline` — 月度趋势 + 连续产出天数(streak) + 速度(velocity)
- `GET /api/analysis/coverage` — 跨框架覆盖率汇总
- `GET /api/analysis/gaps` — 盲区检测（未覆盖方法论 + 过期维度 + 知行比）
- `GET/POST/PUT/DELETE /api/analysis/reports` — AI 报告 CRUD，落盘到 `<dataDir>/analysis/reports/`

**GUI — 成长档案"认知洞察"tab**（GrowthPage 双 tab + macOS 风格分段控制器）：
- `AnalysisView.tsx` — 四张统计卡片（维度热力图 / 认知速度 / 跨框架覆盖率 / 盲区警示）+ 报告生成按钮 + 历史报告列表
- `ReportViewer.tsx` — 报告生成/查看器（phase 状态管理：generating→completed→error，参考笔记"提炼"缓存机制）
  - React StrictMode 兼容（mountedRef 重置）
  - 三级 JSON 修复（闭合括号 / 截断 narrative / 正则提取）
  - 重试按钮 + 错误诊断信息
  - 报告内容展示：总评 → 认知雷达 → 优势(绿) → 盲区(黄) → 建议(紫) → 详细叙述(markdown)

**LLM 分析服务**（`src/lib/analyzeReport.ts`）：
- 双分支调用（Anthropic SDK + OpenAI-compatible fetch）
- 不设 max_tokens 限制（由 prompt 控制输出篇幅）
- 120 秒超时 + fetch 异常分类捕获（CORS/网络/超时）
- 数据快照压缩（去 crossMatrix，截断盲区列表）
- Prompt（`scripts/ingest/prompts/analysis-report.md`）：篇幅指引 + 6 维度雷达评分标准 + 字段顺序（narrative 放最后防截断）

**前端类型**（`src/types/index.ts`）：`DimensionAnalysis` / `TimelineAnalysis` / `CoverageAnalysis` / `GapAnalysis` / `AnalysisReport` / `AnalysisReportResult` / `RadarDimension`

**前端 API 封装**（`src/lib/dataApi.ts`）：`analysisApi`（dimensions/timeline/coverage/gaps + reports CRUD）

### 设计决策记录
- Tab 命名"认知洞察"（非"AI 分析"，更有成长感）
- 报告按需手动触发，结果落盘（不自动生成）
- 旧报告 generating 状态自动识别为 failed，显示重试按钮

## 5 · ✅ 导师模式 Skill (P1 已交付)

### 已完成内容

**CLI 命令**（`scripts/atomsyn-cli.mjs`）：
```bash
atomsyn-cli mentor [--range week|month|all] [--format data|report]
```
- `--format data`：输出 JSON（dimensions + timeline + coverage + gaps），供 Skill 消费
- `--format report`：输出 markdown 报告（总体画像 + 活跃角色 + 框架覆盖率 + 盲区警示）

**Skill 定义**（`skills/atomsyn-mentor/SKILL.md`，226 行）：
- **混合模式**：Phase 1 数据收集 → Phase 2 单轮报告 → Phase 3 多轮追问
- **教练闭环**：分析 → 建议 → 深入对话 → atomsyn-write 沉淀 → 下次盲区减少
- **与 read/write 的协作表**：6 种对话信号 → CLI 动作映射
- **触发条件**：显式（复盘/mentor/导师模式）+ 静默提示（14 天未复盘时）
- **教练沟通风格**：数据驱动、诚实直率、可执行建议、鼓励不谄媚

**安装同步**：`atomsyn-cli install-skill --target claude` 同时安装 write/read/mentor 三个 Skill

### 与 GUI 的关系
- CLI mentor 和 GUI 的"成长档案 > 认知洞察" tab 共享同一套分析引擎（`scripts/lib/analysis.mjs`）
- CLI 是 Agent 自身分析 + markdown 输出（轻量、即时、可对话）
- GUI 是调用用户配置的 LLM + 持久化保存 + 雷达可视化（更深入、可回看）

## 7 · ✅ 认知雷达 — 轻量版已交付 (P1 已交付，完整版待书架)

### 已完成内容（轻量版，集成在 §3 认知洞察报告中）

**6 维度评分模型**（LLM 基于数据评分 0-100）：

| 维度 | 数据来源 | 评分逻辑 |
|------|----------|----------|
| 认知深度 | 洞察类型分布 + 方法论关联度 | "原则提炼"+"方法验证"占比越高越深 |
| 认知广度 | 覆盖角色/场景/框架数量 | 多角色(≥3)多框架(≥2有覆盖)得高分 |
| 实践转化 | 知行比 + 覆盖率 | 知行比>3且覆盖率>50%得高分 |
| 反思频率 | "情绪复盘"+"原则提炼"占比 | 占比>30%得高分 |
| 学习活跃 | 近30天速度 + streak | 近30天>15条且streak>7天得高分 |
| 理论储备 | 方法论总数 + 框架数量 | >100方法论+3框架得高分 |

**实现方式**：
- LLM prompt 中要求输出 `radar` 字段（固定 6 维度，每维度 score + description）
- `RadarChart.tsx`（纯 SVG + Framer Motion）：六边形网格 + 数据多边形 + 动画
- 概览页（GrowthPage）展示最新报告的雷达快照（替换原心理自查趋势模块）
- 报告详情页展示当次报告的雷达图 + 6 维度详情

**类型定义**：`RadarDimension { axis: string; score: number; description: string }`，`AnalysisReportResult.radar?: RadarDimension[]`

### 完整版待实施（需书架模块数据）
- "理论储备"维度纳入书籍知识点数量
- 独立"认知画像"一级页面（与方法库/记忆花园同级）
- 历史雷达对比（成长轨迹可视化）
- 演进建议可直接跳转方法论/书籍详情页

## 13 · 数据主权原则 (重要)

- 用户数据**只存**平台目录（macOS: `~/Library/Application Support/atomsyn/`）
- 项目 `data/` 目录仅作为**种子源**
- dev 模式和 Tauri 模式共享同一个 `resolveDataDir()` 解析逻辑
- 种子同步机制确保已有用户升级时只添加新文件、不覆盖用户修改
- 分析报告存储在 `<dataDir>/analysis/reports/*.json`
