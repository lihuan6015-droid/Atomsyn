# Atomsyn V2.3 · Bootstrap Tools (agentic + 多文档解析)

> **版本**: V2.3 (proposed, 实施中)
> **状态**: F 组文档同步 (本文档归属)
> **上游**: `docs/plans/v2.2-bootstrap.md` (V2.2 v1 已交付待归档) · `openspec/changes/2026-04-bootstrap-tools/`
> **目的**: 让任意会话不读 spec 也能理解"V2.3 在 V2.2 之上做了什么、为什么、怎么用"

---

## 1 · 一句话叙事

V2.3 把 V2.2 的 bootstrap **从 demo-friendly 升级到硬盘真实可用**: (i) 加 `.docx` + `.pdf` 解析 (mammoth + pdfjs-dist), (ii) 在聊天对话流里通过 `/bootstrap` 命令 + 路径粘贴智能识别启动, (iii) 用 LLM tool-use loop + Agent 工具集 (`ls / stat / glob / grep / read`) 替代 v1 硬编码 funnel —— 让 LLM 探索式处理 4400+ 文件 / 中文目录 / 代码 + PDF + 笔记混杂的真实硬盘。

---

## 2 · 为什么需要 V2.3

V2.2 (bootstrap-skill) dogfood 暴露三个核心痛点 (用户在 `~/Documents/开发过程资料` 跑首跑遇到):

1. **GUI 文档选择器单一**: BootstrapWizard 第一屏只能选单目录, 不能选具体的 `.md / .docx`。"我想只导入这 5 个 PRD" 做不到, 只能选父目录让 funnel 全扫
2. **对话流缺入口**: 用户在 ChatPage 想触发 bootstrap, 必须中断对话流去找"初始化向导"按钮 (7 步操作链)
3. **硬编码 funnel 处理混合目录效果差**: 真实"开发过程资料"是 4437 文件 / 9 个中文子目录 (10月材料 / Langgraph调试日志 / UI设计 / 竞赛材料 / 我的研究 / ...) + 大量 `.py / .pyc / .jpg / .pdf` 混杂。v1 funnel 不区分 "Langgraph调试日志" (技术日志, 高价值) vs "10月材料" (混杂, 中价值) vs `.pyc 文件` (零价值)

更隐蔽的两个:
4. 用户的 PRD / 调研报告大量是 `.docx`, 论文是 `.pdf` —— v1 全跳过, "导入存量素材"承诺打折
5. 即使 GUI 选了 `~/Documents/X`, packaged Tauri 模式因为 `fs:scope` 不含 `~/Documents` 报权限错

V2.3 的目标是兑现 V2.2 北极星里"导入存量"的真实可行性 —— 让"两个月前的顿悟会安静地出现"在用户硬盘真实目录上也能跑通, 不是 demo-only。

## 4 · 关键架构决策 (与 V2.2 的 delta)

### 4.1 agentic vs funnel (D-001 + D-008)

**默认 agentic** (LLM tool-use loop), funnel 留作 fallback。理由:
- v1 funnel 在真实混合目录效果差是已知 bug, 默认它就是默认 bug
- agentic 是真正解决方案, 但 LLM tool-use 本身有不确定性 → 留 funnel fallback 保守
- 用户不需要为新功能学新 flag, 体感无门槛升级
- agentic 失败时, `cmdBootstrap` 自动 fallback funnel 一次 + WARN, 不让用户卡死

### 4.2 Agent 工具集 + path-prefix 沙箱 (D-006)

5 个原子工具暴露给 LLM:
- `ls(path)` — 列目录 ≤ 200 entries
- `stat(path)` — 大小/mtime/类型
- `glob(pattern, root)` — 匹配文件 ≤ 500 (支持 `*` `**` `?` `{a,b,c}`)
- `grep(pattern, file)` — case-insensitive regex ≤ 50 hits, ≤ 16 KB scan
- `read(file)` — 走 extractors/ + privacy 链, ≤ 16 KB 输出 (强敏感整文件 skip, 弱敏感 redact)

所有路径必须 `startsWith` 用户给的 `--path` (sandboxRoots), 越界抛 `SANDBOX_VIOLATION`。macOS HFS+ 的中文文件名 NFD 标准化在沙箱内部 normalize 到 NFC, glob 不会因为 Unicode form 错过中文目录。

### 4.3 双重上限 (D-009)

LLM tool-use loop 同时受:
- `maxLoops = 30` (一轮 ≈ 一次 LLM 调用 + N 次工具)
- `maxTokens = 100_000` 累计 input + output

任一触发即终止, 输出 partial-result markdown。100k token ≈ Claude Sonnet $0.30, 用户预算可控。

### 4.4 chatWithTools 双分支 (D-005)

`scripts/lib/bootstrap/llmClient.mjs` 加 `chatWithTools()` 方法, 双分支实现 Anthropic `tool_use` content block + OpenAI `tool_calls`, 归一化为统一 `{ stop_reason, text, toolCalls, usage }` 返回 — agentic.mjs 不感知 provider。

### 4.5 文档解析层 (D-004)

`scripts/lib/bootstrap/extractors/` 目录拆分:
- `markdown.mjs` — 含 frontmatter peel
- `code.mjs` — 大文件 head + tail 截断
- `text.mjs` — 通用 UTF-8 (含 binary detection)
- `docx.mjs` — mammoth.extractRawText (lazy import)
- `pdf.mjs` — pdfjs-dist legacy ESM build, 前 5 页 + 文档元信息 (lazy import)
- `index.mjs` — 统一调度 + privacy.scanText/redactWeakInText 链

`mammoth + pdfjs-dist` 选型: 纯 JS, 0 外部依赖, Tauri 打包内可用; bundle 增 ~2 MB 可接受。

### 4.6 入口扩展

- ChatInput SkillCommandPalette 加第 4 个命令 `/bootstrap` (Sparkles icon)
- ChatInput.handleSend 拦截 `/bootstrap [path]` → 派发 `atomsyn:open-bootstrap` 事件
- ChatInput.onPaste 检测绝对路径 → 弹出 `<PathDetectionBanner>` (D-002, 带 dismiss + "不再提示" 30 天 localStorage)
- ChatPage 监听事件 → 打开 BootstrapWizard + 预填 path
- BootstrapWizard PathsScreen 加多选目录 + 多选具体文件 (filter `.md/.markdown/.txt/.docx/.pdf/.json/.yaml`)
- BootstrapWizard DryrunScreen 加 agent_trace timeline 折叠面板 (默认折叠, 节省视觉)

### 4.7 Tauri scope 扩展 (D-007)

`src-tauri/capabilities/default.json` fs:scope 加:
- `$HOME/Documents` + `$HOME/Documents/**`
- `$HOME/Downloads` + `$HOME/Downloads/**`
- `$HOME/Desktop` + `$HOME/Desktop/**`

**不开** `~/**`, 守住隐私边界 (避免前端代码读 `~/.ssh / ~/Library/Cookies / 浏览器配置`)。用户笔记在非默认位置 (e.g. `~/Dropbox`) 仍可通过 CLI 模式跑 (CLI 不受 Tauri scope 限制)。

## 5 · 不可变契约 (Iron Promises 增量)

V2.2 的 B-I1 ~ B-I8 全部继承不变。V2.3 新增:

- **B-I9 · agentic 沙箱**: agentic 模式下所有 `ls/stat/glob/grep/read` 路径必须落在 `sandboxRoots` 内, 越界拒绝。`read` 走 extractor 链 + privacy 链, 不绕过。
- **B-I10 · agentic loop 双重上限**: `maxLoops=30 + maxTokens=100k`, 任一触发即终止。失败自动 fallback funnel 一次 + WARN。

---

## 6 · 与 V2.2 的兼容性

- v1 用户的 session JSON 加载零异常 (additive `options.mode` / `agent_trace[]` 缺失时 fallback `mode='funnel'` + `agent_trace=[]`)
- v1 `--mode funnel` 等价于 v2 `--mode funnel`, 输出与 v1 完全相同
- v1 的 commit prompt (`scripts/bootstrap/prompts/commit.md`) 在 v2 不变 — agentic 模式输出严格按 v1 markdown 模板, commit 阶段不感知是哪种模式产出
- BootstrapWizard 5 屏 UI 不破: 加了多选 + agent_trace timeline 是 additive

## 7 · 验证 / Dogfood

主 dogfood 目标: `~/Documents/开发过程资料/` (4437 文件 / 9 个中文子目录 / `.py + .pyc + .jpg + .txt + .json + .pdf` 混杂)。

期望 (G7 任务):
- ≥ 80% 候选来自 `.md / .txt / .docx / .pdf`, 不含 `.pyc / .jpg`
- agent_trace 中 LLM 优先选 `我的研究` / `Langgraph调试日志` 等高语义子目录
- 总耗时 ≤ 45 min (v1 funnel 估 130 min 基线)
