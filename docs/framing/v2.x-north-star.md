## 0 · 为什么需要更新北极星

V2.0 的北极星——"把你的认知资产，从 AI 里拿回来"——回答的是**主权和存储**问题。V2.0 M0-M6 交付后，产品的重心正在从"沉淀 + 唤醒"双循环转向**"沉淀 → 结构化 → 反思 → 行动 → 再沉淀"五段闭环**。旧北极星只覆盖了前两段。

V2.x 的 8 个新方向（方法库泛化、统计可视化、AI 复盘、书架、导师模式、聊天改进、认知雷达、新学习范式闭环）要求北极星能同时承载**仓库、结构、教练**三层价值。

## 1 · 产品定位

### 一句话定位

**Atomsyn 是一个认知的双向操作系统。**

底层是仓库（沉淀 + 结构化 + 唤醒），上层是教练（反思 + 行动引导），两层互相喂养。

### 三层架构

```
┌─────────────────────────────────┐
│  上层 · 教练层 (Coach)           │  ← 主动引导：看见盲区、推动复盘、建议行动
│  导师模式 · 认知雷达 · AI 复盘    │
├─────────────────────────────────┤
│  中间 · 结构层 (Structure)       │  ← 让认知"长出骨架"：方法论 × 碎片相遇
│  双骨架 · 书架提炼 · 统计可视化   │
├─────────────────────────────────┤
│  底层 · 仓库层 (Vault)           │  ← 沉淀 + 唤醒：CLI ingest / read / 笔记
│  Agent CLI · 笔记模块 · 记忆花园  │
└─────────────────────────────────┘
```

- **仓库层** (V2.0 已交付): 把散落的认知碎片收进来，存得住、找得到、AI 能调
- **结构层** (V2.x 中间层): 让碎片长出骨架，和方法论相遇，从"一堆碎片"变成"一张地图"
- **教练层** (V2.x 上层): 主动推用户复盘、指出盲区、建议下一步行动，对抗人性的惰性

三层互相喂养：仓库为结构提供原料，结构为教练提供数据，教练推动用户产出新碎片回流仓库。

## 2 · 北极星短语 (V2.x)

### 主句

> **让你积累的认知，在需要时醒来。**

### 完整价值主张

> **让你积累的认知，在需要时醒来。**
>
> 方法论、踩坑、顿悟、阅读——它们没有消失，只是沉睡在你的记忆角落。Atomsyn 把它们变成有结构、会生长的认知地图。
>
> 它会告诉你哪些领域已经扎实、哪些还是空白，在你懒得回头看的时候轻轻推你一把。
>
> 而当你下一次和 AI 对话时，你曾经的某个顿悟会安静地出现在回答里——不是被搜索出来的，是它自己找到了你。

### Tagline

> *Atomsyn — it remembers, so you can grow.*

### 三段 × 三层对应关系

| 段落 | 对应层 | 核心承诺 |
|---|---|---|
| "沉睡...变成有结构、会生长的认知地图" | 仓库层 + 结构层 | 你的认知不会丢，还会长出骨架 |
| "告诉你盲区...轻轻推你一把" | 教练层 | 系统主动推你成长，对抗惰性 |
| "顿悟会安静地出现...是它自己找到了你" | 相遇层 (仓库 + 结构的产物) | 在对的时刻，过去的认知和当下的需求相遇 |

## 3 · 价值主张演进全景

| 版本 | 核心问题定义 | 北极星主句 | Tagline | 产品隐喻 |
|---|---|---|---|---|
| **V1** | 方法论学过即忘 | "认知底气重建工具" | — | 知识图书馆 + 演练场 |
| **V1.5** | 个人记忆资产无法被自己和 AI 共同检索调用 | "主权元认知层" | — | L1 + L2 双层主权架构 |
| **V2.0** | 方法论与经验碎片永不相遇 | "把你的认知资产，从 AI 里拿回来" | *your meta-cognition, rematerialized* | 双骨架 + 双向蒸馏相遇 |
| **V2.x** | 认知沉淀了但不会自动成长 | **"让你积累的认知，在需要时醒来"** | ***it remembers, so you can grow*** | **认知双向操作系统 (仓库 + 结构 + 教练)** |

### 演进逻辑

```
V1   "记住方法论"         → 解决了"存"
V1.5 "让 AI 也能调用"     → 解决了"取"
V2.0 "让方法论和经验相遇" → 解决了"连"
V2.x "推你用它们成长"     → 解决了"长"

存 → 取 → 连 → 长
```

每个版本都不是否定前一版，而是在前一版的地基上长出新的一层。

## 4 · 面向用户的沟通要素 (Landing Page 取材)

### 4.1 一句话电梯演讲

> Atomsyn 把你散落的方法论、踩坑经验和阅读顿悟，变成一个会生长、会在对的时刻唤醒你的本地认知系统。

### 4.2 三个核心卖点 (Feature Pillars)

| # | 卖点 | 一句话 | 对应层 |
|---|---|---|---|
| 1 | **沉淀即投资** | 一条命令、一段笔记，你的每次思考都被结构化地留住 | 仓库层 |
| 2 | **看见你的认知地图** | 哪些领域扎实、哪些是空白，一眼看清自己的成长轨迹 | 结构层 + 教练层 |
| 3 | **在对的时刻相遇** | 和 AI 对话时，你两个月前的顿悟会安静地出现在回答里 | 相遇层 |

### 4.3 差异化定位 (Why not X?)

| 竞品类型 | 它们做了什么 | Atomsyn 多做了什么 |
|---|---|---|
| Notion / Obsidian | 存笔记，人去翻 | 笔记会自己在 AI 对话里找到你 |
| ChatGPT Memory / Claude Memory | AI 记住你，但数据在云端 | 100% 本地主权，数据永远是你的 |
| Mem / Rewind | 全量记录，没有结构 | 双骨架结构 + 方法论关联 + 教练引导 |

### 4.4 北极星 Demo 场景 (继承自 V2.0 framing，扩展教练层)

> 两个月前的一个下午，你在访谈后随手记了一段踩坑笔记。一条命令，Atomsyn 把它结构化存好，悄悄挂到了 JTBD 方法论下。
>
> 一个月后，Atomsyn 的认知雷达告诉你："你在'用户访谈'领域积累了 23 条经验，但'竞品分析'还是空白。"你想了想，决定下个项目有意识地补这块。
>
> 又过了一个月，你在 Cursor 里起草新项目的访谈提纲。Atomsyn 的回答里除了方法论要点，还多了一小段：
>
> > 📎 **你两个月前记下过**："访谈对象嘴上说的'最缺'不一定是真'最痛'，下次要在 15 分钟后再问一次。"
>
> 你愣了两秒，然后把这条写进了提纲。
>
> **存 → 看见 → 行动 → 相遇。这就是认知复利。**

## 5 · 本文档对下游的约束

1. **V2.x 所有新功能在进入 PRD 前，必须先回到本文档 §1 的三层架构检查**：它属于哪一层？它如何喂养其他层？
2. **Landing Page / 宣传页面从 §4 取材**，不得脱离本文档的核心叙事框架
3. **Tagline "it remembers, so you can grow" 是品牌级资产**，在所有外部露出中保持一致
4. **V2.0 的旧北极星 "把你的认知资产，从 AI 里拿回来" 不作废**，降级为仓库层的 sub-tagline，在强调本地主权时使用

## 6 · 不可动摇的哲学 (继承 + 扩展)

以下继承自 V2.0 framing §5，新增第 7-8 条：

1. **本地 100%，主权第一** — 用户数据从不离开本地
2. **L1 + L2 双层缺一不可** — 三层架构必须在 GUI 和 CLI 上同时成立
3. **大厂结构性不会做** — 教练层要求放弃云端记忆模型，和大厂商业模式直接冲突
4. **使用即维护** — 碎片被 ingest、被相遇、被复盘的那一刻，就是它被维护的那一刻
5. **CLI-first** — 所有写入必须经过 `atomsyn-cli`
6. **设计语言硬约束** — Linear/Raycast 变体 + Inter + 玻璃态 + spring + 渐进披露
7. **教练不居高临下** — "轻轻推你一把"，不是"教你怎么做"。镜子，不是老师
8. **相遇不炫技** — 安静的、可被忽略的、但一旦被注意到就无法忽视的
