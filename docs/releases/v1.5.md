# CCL Atlas v1.5 · Release Notes

**发布日期**: 2026-04-08
**代号**: 主权元认知层 · Sovereignty Meta-Cognition Layer

## 一句话概括

CCL Atlas 从"一个本地方法论图书馆"升级为**一份真正可以被 AI 调用的、数据 100% 在你本地的主权元认知层** —— 你用 Claude Code 或 Cursor 时,可以让 AI 把学到的东西沉进来、也可以让下一次 AI 回答前先读一读你过往的积累。

## 新增能力

### 1 · Agent 双向接口 (Gap-2 主角)

**双 skill 契约**: `atlas-write` 和 `atlas-read` 安装到 `~/.claude/skills/` 与 `~/.cursor/skills/`,触发词全中文 + 英文混合,覆盖普通用户和开发者两种表达:

- **写入**: 帮我记下来 / 这个要记住 / 存到我的 atlas / 存到 CCL / 加到 CCL Atlas / 加到我的知识库 / 以后还会用到 / 把这个经验保存一下 / CCL 记一下 / 别让我忘了 / save to my atlas / save to CCL / remember this for me
- **读取**: 我之前学过类似的吗 / 我的知识库里有没有 / CCL 里有没有 / 从 CCL 里找 / 看看我之前记过什么 / 调用我的经验 / 我以前是怎么做的 / 查一下 atlas / check my atlas / check CCL

**CLI-first 架构**: Skill 不再让 agent 手写 JSON。Agent 只负责**从对话里摘取用户视角的字段**(name / sourceContext / insight / tags),通过 stdin 送进 `atlas-cli write --stdin`。所有 schema 合规性、id 生成、slug 派生、时间戳、索引重建、锁冲突检测、usage 日志 —— **全部由 CLI 承担**。这保证了即使 schema 演进、数据目录搬家,写入契约也稳定。

### 2 · atlas-cli 命令行

零依赖 Node ESM CLI,接口:

```
atlas-cli write --stdin            # 从 stdin 读松耦合 JSON,写入 experience atom
atlas-cli read  --query "..."      # 检索并返回 markdown
atlas-cli reindex                  # 重建知识索引
atlas-cli where                    # 打印当前数据目录 + 来源
atlas-cli install-skill            # 把 skills 装进 ~/.claude/skills + ~/.cursor/skills,并安装 CLI shim
```

**install-skill 会**:
1. 拷贝 `skills/atlas-write` 和 `skills/atlas-read` 到目标 agent 目录
2. 在 `~/.ccl-atlas/bin/` 写一份 shell shim (macOS/Linux) + `.cmd` shim (Windows)
3. 打印 PATH 追加指令,由用户决定是否加入 shell rc

### 3 · Tauri 桌面壳 + 首次运行 seeding

- Tauri v2 完整脚手架(dmg/app/msi/nsis 四平台打包目标)
- 4 个 Rust 初始化命令: `init_ensure_data_dir` / `init_seed_frameworks` / `init_seed_methodology` / `init_check_skill_installation`
- 首次运行自动从 bundled resources 拷贝 frameworks + 125 条 methodology atoms 到用户数据目录
- 非破坏性: 已有文件**绝不覆盖**,尊重用户编辑
- 跨平台数据目录解析: `CCL_DEV_DATA_DIR` env → `~/.ccl-atlas-config.json` → 平台默认 (macOS `~/Library/Application Support/ccl-atlas`, Linux `~/.local/share/ccl-atlas`, Windows `%APPDATA%/ccl-atlas`)

### 4 · 启动加载屏 (Splash)

- 首次进入(每 session 一次)显示品牌 Splash
- 4 步初始化进度条: 准备数据目录 → 初始化骨架 → 加载图书馆 → 检测 Agent Skill
- 玻璃态 / violet-sky-emerald 渐变 / spring 动画 / 支持 `prefers-reduced-motion`
- 锁定深色"loading veil"避免主题切换闪烁
- 出错时可点"重试"

### 5 · Skill 地图 Tab (Gap-4)

- 新顶部 Tab `/skills`,展示本地所有 AI skill(扫描 `~/.claude/skills/`, `~/.cursor/skills/`)
- 4 层渐进披露卡片: L0 基础信息 → L1 hover 显示元数据 → L2 点击展开详情 → L3 操作按钮
- 筛选: 工具 × 标记状态(常用/已遗忘/未使用) × 搜索
- **AI 摘要生成**: 点击"✨ 让 AI 理解这个 skill"调用 Anthropic SDK 生成 `aiGeneratedSummary` / `aiGeneratedTags` / `typicalScenarios` / `triggerKeywords`,按需触发不批量

### 6 · Agent 活动 Feed (Gap-6.1)

- 知识图书馆页面右侧抽屉,56px 头部,glass 背景
- 从 `usage-log.jsonl` 聚合 write/read 事件,newest first,top 50
- 筛选: action / 时间窗 / agent 名
- 每条可展开看 query / sourceContext / 返回数量
- 点击 atomId 跳转到对应原子详情页

### 7 · Atom 校准 · 调用历史 (Gap-6.2)

- Atom 详情页新增"🔗 调用历史"折叠区: 该原子被哪些 agent 调用过、最近一次、query 列表
- 三个校准按钮:
  - **合并**: 选兄弟原子 → V1.5 stub(记录 usage intent, V1.6 正式实现)
  - **降权**: 翻转 `stats.userDemoted`,atlas-cli read 会自动过滤
  - **锁定**: 翻转 `stats.locked`,atlas-write 拒绝覆盖
- 原子被锁后,标题栏显示琥珀色锁定徽章

### 8 · Agent 权限 Settings (Gap-6.3)

- 设置页新增"🤖 Agent 权限" section
- 从 usage-log 自动发现已调用过的 agent
- 每 agent 三开关: 读权限 / 写权限 / 写入需确认
- V1.5 暂存 localStorage,并明确标注"V1.5 UI 框架, V1.6 Tauri 桌面版写入 ~/.ccl-atlas-config.json"
- 支持手动添加 agent / 删除

## V1.5 终结增量 · Session 4 收尾 (2026-04-09)

> 上面的章节写于 V1.5 主干 ship 当晚 (2026-04-08)。第二天 dogfood 暴露出一批"看起来对但运行不 work"的 bug,以及若干用户体验上的 gap。这一节追加记录所有补丁,包括架构层面的两次根因发现 (Tauri 数据目录分叉、macOS 红绿灯 hit-test) 和 atlas-cli 的一次重要扩张 (update / find / get)。

### 一、atlas-cli · 三个新子命令

为了让 agent 能**安全更新**已有 experience atom 而不是只能 write+rm 制造冗余:

| 命令 | 用途 |
|---|---|
| **`update --id <id> --stdin`** | 按 id 定位现有文件 (无论在哪个 slug 目录) → merge caller 字段到 existing → 如果新 name 产生新 slug → 原子迁移 (写新 slug + 删旧文件 + 旧目录空就 rmdir) → append usage log。**冷启动新会话也能安全调用**,不需要任何前置清理 |
| **`get --id <id>`** | 打印单张 atom 完整 JSON,exit 2 表示未找到 (供 agent 做存在性判断) |
| **`find --query "..."`** | 关键词搜索,返回 `{id, name, tags, path, score}` JSON 数组 |

### 二、双 skill 升级 · 新建 vs 更新双模式

`atlas-write` SKILL.md 不再只是"创建新 atom",新加 Step 0 决策表:

- **🆕 新建** (原触发词): "帮我记下来 / 存到 CCL / save to my atlas / ..."
- **🔄 更新** (新触发词): "更新那条经验 / 补充到之前那条 / 把这个也合进 X 里 / 修正刚才的 / 合并进旧的 / update that atom / merge into the existing one"

更新流程: agent 必须先 `atlas-cli find` 拿到 id,再 `atlas-cli update --id <id> --stdin`,**绝对禁止 write + 手动 rm 旧文件** (反模式已写入 SKILL.md)。

`atlas-read` SKILL.md 也升级: 强调返回的 atom id 可以**直接喂给 atlas-write 的更新模式**,实现 read → update 的无缝 chaining。

### 三、Tauri 桌面集成 · 三连坑修复 (一次性沉淀)

完整经验沉淀在 atom `atom_exp_macos-tauri-hit-test-vs-drag-region_1775656757`:

**坑 1 · `titleBarStyle: "Overlay"` 的 hit-test 冲突**
- 现象: 拖一次窗口就再也拖不动,鼠标移出窗口让红绿灯 fade-out 后又能拖一次
- 根因: macOS 保留整个 28px title bar 区域的 hit-test 所有权,红绿灯 visible 时强行接管所有 mousedown,前端 `startDragging()` 收不到事件
- 修复: 删除 `titleBarStyle: "Overlay"`,让 macOS 画 native 28px title bar (`title: ""` + `hiddenTitle: true` 隐藏文字)。内容从 y=28 开始,native title bar 由系统接管 drag/red-light/double-click,完全可靠

**坑 2 · Native title bar 跟系统外观而非 app 主题**
- 现象: app 是浅色但 macOS 系统是深色 → title bar 黑色,和 app 强烈割裂
- 修复: `tauri.conf.json` 加 `theme: "Light"` 作初始 + `useAppStore.setTheme` 调 `getCurrentWindow().setTheme(t)` 实时同步 + `onRehydrateStorage` 首屏 sync 一次防闪烁

**坑 3 · Tauri v2 permission deny-by-default 而且静默失败**
- 现象: 坑 2 写完代码后 setTheme 依然不生效
- 根因: `core:window` permission 没在 `capabilities/default.json` grant,前端调 → Tauri 默默拒绝 → promise reject → 被 `.catch(() => undefined)` swallow
- 修复: capabilities 补 `core:window:default` + `core:window:allow-set-theme` + `core:window:allow-start-dragging` + `core:window:allow-theme`

**调试规则 (写进 keySteps 的元教训)**:
1. 凡是前端调 Tauri window API 没反应,**第一步不是查 JS,是查 capabilities permission**
2. 不要 `catch(() => undefined)` 掉 Tauri promise 错误,deny-by-default 拒绝本来就隐身
3. 检查顺序: ① capabilities → ② tauri.conf windows → ③ 前端 JS → ④ React race。**80% 问题卡在前两层**

### 四、数据目录分叉 · 上半夜被忽略的根因

V1.5 主干 ship 时 vite 插件硬编码 `dataDir = path.resolve(__dirname, 'data')`,只在项目 /data/ 下读写。Rust 端用 `app.path().app_data_dir()` 解析到 `~/Library/Application Support/com.circlelee.ccl-atlas/ccl-atlas/`,atlas-cli 则解析到 `~/Library/Application Support/ccl-atlas/`。**三端各说各话**:

- GUI 看不到 atlas-cli 写的 experience atoms (在不同目录)
- Splash 显示"未找到打包的方法论原子资源" (Rust 看错地方)
- seed-check 永远不弹 (用错路径对比)

**修复**: 三端**完全镜像**同一个解析链:
```
1. $CCL_DEV_DATA_DIR  (env override · dogfood mode)
2. ~/.ccl-atlas-config.json  dataDir 字段
3. 平台默认 (macOS: ~/Library/Application Support/ccl-atlas, Linux: ~/.local/share/ccl-atlas, Windows: %APPDATA%/ccl-atlas)
```

Rust 不再用 `app.path().app_data_dir()` (它会自动加 bundle id 前缀),改用 `home_dir() + cfg!(target_os)` 手工构造,和 vite + atlas-cli 字符级别一致。

vite 插件 `seedDataDir()` 在拷贝完成后立即写一份初始 `.seed-state.json` (含 sha256 manifest),让 seed-check 端点的版本对比有可用基线。

### 五、Settings 白屏的两层防御

实测发现 `config/llm.config.json` 存在合法但不完整的状态,渲染时崩在 `config.providers[undefined].xxx` / `config.copilot.xxx`。修复:

- **`useEffect` 加载阶段做整体 normalize**: 把 `llmConfigApi.get()` 的返回值结构化重建,**所有字段强制存在**,缺失的字段填写默认值 (anthropic / openai / custom 三个 provider 各自的默认 model + url + tokens,copilot 的 systemPromptRef + maxContextAtoms + flags)
- **render 早期再加一层降级**: `provider` 三级 fallback (config.activeProvider → first enabled → 'anthropic'),`providerCfg ?? { ... }` 兜底
- **`tsconfig.node.json` 缺 `target: ES2022`** → `tsc -b` 在 vite 插件的 `Set` 迭代上报错 → `npm run build` 失败 → tauri 用 stale dist → 各种诡异行为。补 target 后 build 干净

### 六、Splash 浅色重做 + 默认浅色主题

- 默认主题从 `dark` 改为 `light` (`useAppStore` initial state + `index.html` pre-paint script)
- Splash 不再"锁定深色 veil",通过 `document.documentElement.classList.contains('dark')` 实时读取主题,所有色值改 `isDark ? ... : ...` 双分支
- `InitStepRow` 全部 `text-emerald-600 dark:text-emerald-400` 风格补浅色对照
- `autoAdvance` 默认 `false`,初始化完成后用户**手动点"进入 CCL Atlas"**,不再自动跳
- 加 min-hold 450ms/步,避免一闪而过

### 七、Skill 扫描跟随符号链接

`scripts/scan-skills.mjs::walkSkillFiles` 之前 `Dirent.isDirectory()` 在 symlink 上返回 false,漏掉了 `~/.claude/skills/frontend-design → ../../.agents/skills/frontend-design` 这类。修复: 检测到 `isSymbolicLink()` 时调 `stat()` 解真实路径分类,带 inode-based cycle guard。**扫描从 64 → 98 skills**,多了 30 个真实可用的入口。

### 八、其他散点

- **打开文件夹**: 不走 `tauri-plugin-shell` 的 `plugin:shell|open` (它的 OpenScope 默认阻止任意本地路径),改用 Rust 原生 `open_path(path)` 命令,直接 shell `open / explorer / xdg-open`
- **`vite-plugin-data-api` 加 `_absPath`**: 让前端能在 `it._absPath` 拿到 atom 的绝对路径,喂给 `openContainingFolder()`
- **AppShell 改 `h-screen overflow-hidden`** + 移除自定义 32px drag strip (native title bar 接管)
- **TopNav 改 `bg-white dark:bg-[#0a0a0b]`** 完全不透明 + `z-50`,sticky 时滚动内容不再穿透
- **`backgroundColor: "#ffffff"`** 在 tauri.conf,窗口启动第一帧就是白色 (匹配默认浅色主题)
