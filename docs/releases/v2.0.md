## 元教训

1. **中文关键词匹配有天然噪音**: 单字匹配会产生大量误关联。V2.x 需要探索中文分词或 embedding 方案。但 M4 证明了即使是粗糙的关键词匹配,在方法论→碎片的"相遇"场景中也能产生有价值的关联。

2. **Skill description 是唯一的路由决策入口**: 模型只读 frontmatter description 来决定是否触发 skill。过拟合具体场景不如传达设计哲学 — 让聪明的 Agent 自己判断。

3. **渐进式读取比一次性返回更好**: read→get 两步比直接返回完整内容节省 80% token,且让 Agent 有判断力决定钻取哪些原子。

4. **fragment 和 crystallized experience 的字段不统一**: `title` vs `name`, `summary` vs `insight` 造成了 GUI 渲染的兼容问题。V2.x 应统一字段命名。

5. **findAtomFileById 必须搜索所有目录**: V1.5 只搜 experience/,M4 扩展后需搜全部原子目录,否则 `get` 找不到方法论 atom。
